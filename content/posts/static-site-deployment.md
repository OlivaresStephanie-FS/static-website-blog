Deploying a static website helped me better understand how modern front-end applications are hosted and managed. Before this week, I mostly focused on writing code locally and seeing it run in development environments. Going through the deployment process showed me how important version control, build tools, and hosting platforms are in real-world development.

One of the biggest takeaways for me was learning how services like Vercel connect directly to GitHub repositories. Making changes locally, pushing them to GitHub, and then seeing the live website automatically update made the workflow feel much more professional and practical.

I also learned that deployment is not always as simple as uploading files to a server. Node versions, dependencies, build errors, and configuration files all play a role in whether an application runs correctly online. Even simple projects can run into issues if the environment is not configured properly.
